Problem

Două portofolii paralele

Two parallel portfolios

Aveam zona CRM (Alpis Fusion, ClientFlow, ClientOps) și zona arcade (ARCADE OPS, BASKET VS AI, NEXUS ARCADE) — bine construite separat, dar fără un proiect care să le conecteze. Recruiterii care intrau pe CRM nu vedeau abilitățile de game design și invers.

I had the CRM area (Alpis Fusion, ClientFlow, ClientOps) and the arcade area (ARCADE OPS, BASKET VS AI, NEXUS ARCADE) — both built well separately, but no project bridging them. Recruiters landing on CRM didn't see the game design skills and vice versa.

Solution

Un singur proiect cu trei layere

One project, three layers

NEXUS PROTOCOL pune un endless runner ca shell, un CRM tycoon ca meta-layer și mini-challenges de cod la checkpoint-uri. Modulele CRM nu sunt cosmetice — fiecare schimbă observabil regulile run-ului.

NEXUS PROTOCOL puts an endless runner as the shell, a CRM tycoon as meta-layer, and code mini-challenges at checkpoints. CRM modules aren't cosmetic — each observably changes run rules.

LAYER 01 · DRIVE

Endless runner synthwave

Synthwave endless runner

Trei lane-uri, perspectivă falsă cu soare retro + grid scrolling, obstacole holografice (block / spike), pickup-uri de tip lead, energy și boost. Speed ramp progresiv: 4 → 14 unități pe măsură ce avansezi.

Three lanes, fake perspective with retro sun + grid scrolling, holographic obstacles (block / spike), pickups for leads, energy and boost. Progressive speed ramp: 4 → 14 units as you advance.

LAYER 02 · CRM

Module store între run-uri

Module store between runs

La fiecare 10 lead-uri închise se deschide CRM Module Store. Cumperi module cu credite acumulate. Modulele persistă între run-uri și fiecare schimbă o regulă concretă (vezi tabel mai jos).

Every 10 closed leads opens the CRM Module Store. You buy modules with accumulated credits. Modules persist between runs and each changes one concrete rule (see table below).

LAYER 03 · CODE

Checkpoint-uri cu cod real

Checkpoints with real code

La fiecare 1000m apare un Code Breaker: typeof null, [10,1,2].sort(), 0.1+0.2 === 0.3, regex, CSS specificity. 8 secunde, 4 răspunsuri. Corect → multiplier ×2 timp de 15s. Greșit → slowdown 3s. Diferențiator clar față de alte runner-e.

Every 1000m a Code Breaker appears: typeof null, [10,1,2].sort(), 0.1+0.2 === 0.3, regex, CSS specificity. 8 seconds, 4 answers. Right → ×2 multiplier for 15s. Wrong → 3s slowdown. Clear differentiator vs other runners.

Decision Log

Decision Log

// DECIZII TEHNICE & ALTERNATIVE CONSIDERATE
// TECH DECISIONS & ALTERNATIVES CONSIDERED

1. Single-file delivery vs Vite + React

1. Single-file delivery vs Vite + React

Am ales single-file HTML (74KB) pentru consistență cu ARCADE OPS și BASKET VS AI. Deploy într-un push, fără build pipeline, recruiterul poate citi tot codul într-un singur fișier.

I chose single-file HTML (74KB) for consistency with ARCADE OPS and BASKET VS AI. Deploy in one push, no build pipeline, recruiters can read the entire codebase in one file.

Alternativă: Vite + React modular ca Alpis Fusion. Respins pentru că adăuga 30+ fișiere fără câștig funcțional.
Alternative: Vite + React modular like Alpis Fusion. Rejected because it added 30+ files without functional gain.

2. Canvas 2D vs WebGL/Three.js

2. Canvas 2D vs WebGL/Three.js

Estetica synthwave funcționează cu efecte 2D + perspectivă falsă (vertical lines spre punct de fugă). Canvas 2D îmi dă control complet pe glow și particle effects fără shader-e.

Synthwave aesthetic works with 2D effects + fake perspective (vertical lines to vanishing point). Canvas 2D gives me full control over glow and particle effects without shaders.

Alternativă: Three.js cu road mesh real. Respins — adăuga 600KB și un nivel de complexitate disproporționat.
Alternative: Three.js with real road mesh. Rejected — added 600KB and a disproportionate complexity level.

3. Web Audio procedural vs fișiere .mp3

3. Procedural Web Audio vs .mp3 files

Toate sunetele generate live: lead pickup = sine 880Hz → 1320Hz, crash = sawtooth 80Hz, correct answer = arpeggio C-E-G. Zero asset-uri audio înseamnă repo sub 100KB.

All sounds generated live: lead pickup = sine 880Hz → 1320Hz, crash = sawtooth 80Hz, correct answer = C-E-G arpeggio. Zero audio assets means repo under 100KB.

Alternativă: librărie de SFX free. Respins pentru că adăuga 800KB+ și constraint-uri de licență.
Alternative: free SFX library. Rejected because it added 800KB+ and licensing constraints.

4. Numele modulelor — identice cu Alpis Fusion

4. Module names — identical to Alpis Fusion

KANBAN, RBAC, FLOW BUILDER, BILLING — am păstrat același vocabular ca în CRM-ul meu real. Pentru recruiterul tehnic, conectarea e instantă: aceleași concepte aplicate în context diferit.

KANBAN, RBAC, FLOW BUILDER, BILLING — I kept the same vocabulary as my real CRM. For technical recruiters, the connection is instant: same concepts applied in a different context.

Alternativă: nume neutre game-ish (Turbo, Shield, Magnet). Respins — ar fi rupt legătura tematică între portofolii.
Alternative: neutral game-ish names (Turbo, Shield, Magnet). Rejected — would have broken the thematic link between portfolios.

5. localStorage vs backend pentru leaderboard

5. localStorage vs backend for leaderboard

localStorage = zero costuri hosting, match perfect cu portofoliul. Trade-off: nu există competiție globală, dar adaug AGENT-XXX pseudo-entries pentru atmosferă.

localStorage = zero hosting cost, perfect match with the portfolio. Trade-off: no global competition, but I add AGENT-XXX pseudo-entries for atmosphere.

Alternativă: Firebase / Supabase free tier. Respins pentru complexitate operațională și costuri viitoare.
Alternative: Firebase / Supabase free tier. Rejected for operational complexity and future costs.

6. Bilingv RO/EN cu toggle live

6. Bilingual RO/EN with live toggle

Toate string-urile UI într-un dicționar I18N central. Toggle live fără reload. Limba e salvată în localStorage — vine acasă la următoarea vizită.

All UI strings in a central I18N dictionary. Live toggle without reload. Language is saved to localStorage — comes back on next visit.

Alternativă: doar EN. Respins — bilingv RO/EN e parte din identitatea portofoliului meu.
Alternative: EN only. Rejected — bilingual RO/EN is part of my portfolio identity.

Ce am învățat construind asta

What I learned building this

// LESSONS
// LESSONS

1. Vocabularul contează mai mult decât mecanica. Faptul că modulele se numesc KANBAN și BILLING — nu Turbo și Shield — schimbă complet narativul proiectului. Aceleași 5 linii de cod, alt mesaj către recruiter.

1. Vocabulary matters more than mechanics. The fact that modules are called KANBAN and BILLING — not Turbo and Shield — completely changes the project's narrative. Same 5 lines of code, different message to recruiters.

2. Code breaker e cel mai eficient diferențiator. Endless runners există mii. CRM tycoons sunt rari. Endless runner + CRM + întrebări reale de JavaScript = un singur produs în care fiecare alegere de design spune ceva despre cine sunt ca developer.

2. Code breaker is the most efficient differentiator. Endless runners are thousands. CRM tycoons are rare. Endless runner + CRM + real JavaScript questions = one product where every design choice says something about who I am as a developer.

3. Procedural audio merită investiția. Web Audio API e mai accesibil decât pare. Patru ore investite în sound design procedural au eliminat permanent dependența de fișiere audio din toate proiectele mele viitoare cu jocuri.

3. Procedural audio is worth the investment. Web Audio API is more accessible than it seems. Four hours invested in procedural sound design permanently eliminated my dependency on audio files for all future game projects.
